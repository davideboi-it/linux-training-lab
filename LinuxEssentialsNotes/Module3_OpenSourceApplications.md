# Module 3: Open Source Software

## 1 OSS (Open Source Software)
Software released under a license where the copyright holder grants users the right to study, change, and distribute the software to anyone for any purpose.
- **Collaboration:** Usually developed by multiple people through open collaboration for greater product stability.
- **GPL (General Public License):** A widely used free software license that guarantees end users the freedom to run, study, share, and modify the software.

### OSSW Benefits
Open Source Software offers:
- **Security & Transparency:** Code can be audited by anyone.
- **Affordability:** Low or no licensing costs.
- **Interoperability & Scalability:** Easy to integrate and grow within different infrastructures.
- **Localization:** Community-driven translations and adaptations.

### OSI (Open Source Initiative)
A public benefit organization that promotes the use of open-source software and maintains the **OSD (Open Source Definition)**.

### OSSW Costs and Funding
- **Cost Structure:** Usually free or low-cost to acquire. However, professional services such as technical support, administration, and maintenance may involve costs.
- **History:** In the 1970s, software was primarily produced by academics and the "Hacking Culture" for research, not for profit. It was shared in both Machine Code and Human-Readable Code.
- **Commercialization:** As hardware companies began bundling proprietary software, costs increased. Today, OSSW developers sustain projects through private funding, crowdfunding, donations, and commercial partnerships (advertising/sponsorships).

## 2 Open Source Alternatives (Desktop & Server)

**Desktop Applications:**
- **LibreOffice:** Free alternative to Microsoft 365 (Writer, Calc, etc.).
- **VLC:** Universal media player.
- **GIMP:** Professional image editor (alternative to Photoshop).
- **Kdenlive:** Video editor (alternative to Premiere Pro).
- **Audacity:** Professional audio manipulation tool.

**Server & System Tools:**
- **Wireshark:** Network protocol analyzer.
- **GParted:** Disk partition manager.
- **Timeshift:** System restore and backup tool.
- **Atom / VS Code:** Text editors for scripting and development.
- **PuTTY:** Terminal client used for SSH remote control.

---

### Languages and Development Tools
Linux is a versatile environment for multiple programming formats, especially in **Embedded Systems**.

**Interpreted Languages:**
- **Python:** Highly popular interpretive language used for automation and scripting.
- **JavaScript (.js):** Used for creating dynamic content and web applications.
- **Ruby:** Primarily used for web development.
- **PHP:** Server-side scripting language for databases and web servers.

**Compiled Languages:**
- **Java:** Compiled language widely used in the Android ecosystem.
- **C:** The foundational language for Linux; compiled and high-performance.
- **C++:** An evolution of C, compiled and **Object-Oriented (OOP)**.
- **C# (C-Sharp):** Requires a development framework (like .NET) and is compiled.
- **Go:** Modern Object-Oriented compiled language developed by Google.

**Styling & Formatting:**
- **HTML/CSS:** Used for structure and styling of web content.

## 3 Linux Applications and Package Management

### Software Compatibility
- **Distro Dependencies:** Linux applications generally do not run on every distribution because packages and libraries vary across systems.
- **Repositories:** Unlike Windows (.exe) or Mac (.dmg), Linux users primarily download software from **Official Distribution Repositories**.
- **Package Formats:**
    - **.DEB:** Debian-based archives (used by Ubuntu). Managed by `APT`.
    - **.RPM:** Red Hat Package Manager (used by Fedora, OpenSUSE).
    - **.TAR (.tar, .tgz, .tar.gz):** Universal archives, often used by Arch Linux, Slackware, or for manual source code compilation.
- **Ubuntu PPA (Personal Package Archives):** Specialized repositories that allow users to install software not available in the official channels.
