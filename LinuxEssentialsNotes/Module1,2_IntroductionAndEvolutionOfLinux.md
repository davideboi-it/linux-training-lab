# Linux Essentials - Modules 1 & 2

### Overview
- **Linux** has a Unix kernel.
- **Open Source:** Primarily used for servers (96% of the market). Examples: Ubuntu, Debian.
- **CLI (Command Line Interface):** The Shell.
- **GUI (Graphical User Interface):** Common in Windows/Desktop environments.

---

### Open Source & Licensing
You can modify and help develop open-source software. Most have a **Software License Agreement** made by the designers that dictates what you can modify and redistribute.
**Common Licenses:**
- **GNU** (General Public License)
- **Apache**
- **MIT**
- **Unlicense**

---

### Linux Distributions
A **Distribution (Distro)** consists of a Kernel, Software, and Libraries configuration.
- **Kernel:** A low-level program that functions as a bridge between the user and the PC hardware. It manages memory and core functions.
- Every distro can have a different kernel version, software sets, and different scripts/utilities to launch programs.

---

### Release Schedules
- **Cycle:** Alpha -> Beta -> Release.
- **STS (Short Term Support):** Supported for less time, usually less stable.
- **LTS (Long Term Support):** Better for production, supported for more time, highly reliable.
- **Rolling Release:** Some distros and updates are installed fully and continuously under a rolling schedule (no major version jumps).

---

### Embedded Systems & Android
- **Embedded Systems:** Small microcontrollers with Linux that can control the mechanical side. Mostly used in mechanics, medical fields, etc.
- **Android:** An open-source system acquired and developed by Google. It includes a Linux-based system and dominates the mobile/media sector.

---

# System Requirements and OS Fundamentals

### Requirements
- **System Requirements:** The necessary hardware to run a specific OS or software.
- **HRL (Hardware Requirement List):** A detailed list of all required hardware components.
- **Minimum Requirements:** The absolute lowest hardware specs needed for the software to function.
- **Recommended Requirements:** Suggested hardware specs to optimize performance and user experience.

---

### Installing Ubuntu
To install Ubuntu, you must download the **.ISO image** (a virtual disk file).
**Methods:**
- **Physical:** Burn the ISO to a USB drive or CD/DVD.
- **Virtualization:** Use a **VM (Virtual Machine)** to run Linux inside your current OS without erasing your disk.
- **Dual Boot:** Installing Linux alongside another OS on the same physical disk.

---

### OS Comparison & Security
- **Market Share:** Windows dominates the home and corporate desktop market (90%), followed by Mac (7%) and Linux (2% on desktop).
- **Security:** Windows is a frequent target for malware due to its high usage. Linux is considered more secure because its **open-source** nature allows for transparency and rapid community-driven security patches.
- **Cost:** Windows requires a paid license; Mac OS is free but requires Apple hardware; Linux is free and runs on almost any hardware.
- **Control:** While desktops use GUIs, Linux is primarily managed via **CLI (Command Line Interface)** in server environments.

---

### Preemptive Multitasking
A core OS function where the system uses specific criteria to decide how to allocate CPU time to various tasks. It ensures that no single task can monopolize the system resources.
